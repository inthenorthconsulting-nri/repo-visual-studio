# Portfolio Intelligence (reference)

Use when: the task compares multiple products, asks about cross-product
overlap or gaps, or wants an ecosystem/portfolio deck
(`MASTER_AGENT.md` §2.4).

**Prerequisite**: each product listed has already run `synthesize
capabilities` and `synthesize product-identity` (see the three references
above), producing its own `capability-model.json` and
`product-identity.json`. A directory-level `.rvs/portfolio.yml` lists each
product's id and `artifact_root` — schema in
`docs/portfolio-intelligence.md#rvsportfolioyml`.

**Commands**:

```bash
rvs synthesize portfolio [--allow-partial]
rvs export portfolio-model --output portfolio-model.json
rvs export portfolio-claims --output portfolio-claims.json
rvs export portfolio-decisions --output portfolio-decisions.json
rvs portfolio explain <claim-or-decision-id>
rvs create slides --profile portfolio --audience portfolio
```

**Output**: `.rvs/cache/portfolio-model.json` (products, normalized
capabilities, relationships, dependency graph, overlaps, gaps, operating
model, maturity), portfolio claims, portfolio decisions, and — via `create
slides --profile portfolio` — a `PortfolioPlan`.

**Compatibility gate**: a product is excluded (not silently dropped
without a reason) when it's missing a required artifact, was generated by
an unsupported schema version, its identity/capability files disagree, or
the two files are stale relative to each other. `--allow-partial` continues
with only the compatible products but still throws if zero qualify. Full
rules: `docs/portfolio-intelligence.md#intake-and-compatibility-gate`.

**What it never does**: re-scan a product repository from source, or infer
a relationship from a shared vendor/platform name — a relationship or
capability merge that doesn't clear the evidence bar stays `unresolved` /
omitted, never guessed.

Full technical reference: `docs/portfolio-intelligence.md` (overlaps, gaps,
dependency graph, operating model, maturity, decisions) and
`docs/portfolio-showcase.md` (the 13 `PortfolioSceneType` values, plan
rendering, self-hosting proof).
